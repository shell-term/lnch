# プロジェクト概要

## 基本情報

| 項目 | 内容 |
|------|------|
| **プロジェクト名** | lnch（ランチ） |
| **description** | A TUI multi-process launcher for your dev environment |
| **tagline** | One YAML. One command. All your services. |
| **言語** | Rust |
| **ライセンス** | MIT |
| **ターゲットOS** | macOS / Linux (WSL含む) / Windows |

## 解決する課題

IDEやPCの再起動後に散逸する複数ターミナルのローカルサーバー（フロントエンド、バックエンド、DB、ワーカーなど）や定型コマンドを、プロジェクトごとの設定ファイルとして一元管理し、`lnch` の1コマンドで一括起動・復元する。

## 技術スタック

| カテゴリ | クレート | 用途 |
|----------|---------|------|
| TUIフレームワーク | `ratatui` + `crossterm` | ターミナルUI描画・入力制御 |
| 設定ファイルパーサ | `serde` + `serde_yaml` | YAML設定ファイルのシリアライズ/デシリアライズ |
| 非同期ランタイム | `tokio` | 子プロセスの並列管理・ログストリーミング |
| プロセス管理 | `tokio::process` | 子プロセスの起動・停止・IO制御 |
| シグナル処理 | `nix`（Unix） / `windows-sys`（Windows） | プロセスグループ管理・シグナル送信 |
| CLIパーサ | `clap` | サブコマンド・引数解析 |
| エラーハンドリング | `anyhow` + `thiserror` | アプリケーションエラー・ライブラリエラー |
| ロギング | `tracing` | デバッグ用構造化ログ |

## 用語集

| 用語 | 説明 |
|------|------|
| **タスク (Task)** | `lnch.yaml` で定義された1つのコマンド実行単位 |
| **プロセスグループ (PGID)** | Unixにおけるプロセスのグループ。子プロセスを含めて一括でシグナル送信可能 |
| **リングバッファ** | 固定サイズの循環バッファ。容量を超えると古いデータを上書き |
| **トポロジカルソート** | 有向非巡回グラフ(DAG)の頂点を依存関係に沿って線形に並べるアルゴリズム |
| **グレースフルシャットダウン** | SIGTERM で正常終了を試み、タイムアウト後にSIGKILLで強制終了する方式 |
